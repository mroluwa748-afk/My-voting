Semantic HTML & Structure

Why semantics matter for accessibility & SEO

Key tags

  • <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>
  • Use headings in order: <h1> → <h2> → ...
  • Prefer <figure> + <figcaption> for images
  • ARIA only when necessary — prefer native semantic elements
Example 
<main>
  <article>
    <h2>My article</h2>
    <figure>
      <img src="photo.jpg" alt="Instructor demo"/>
      <figcaption>A demo photo</figcaption>
    </figure>
  </article>
</main>

Advanced Form Controls

Reduce JS by using built-in controls

Useful inputs

  • <input type="email" required autocomplete="on">
  • <input type="date"> <input type="time">
  • <input type="color"> <input type="range">
  • <input type="file" accept="image/*" multiple>
  • <datalist> for suggestions
  • <output> for live calculation
Example form 
<form>
  <label>Email <input type="email" required></label>
  <label>Color <input type="color" value="#f00"></label>
  <label>Quantity <input type="range" min="1" max="10" value="5"></label>
</form>

Media & Responsive Images

Use <picture>, lazy loading, and accessible media

Techniques

  • <picture> + <source> for art direction
  • <img loading="lazy" alt="..."> to improve perf
  • <video controls muted playsinline poster="...">
  • Provide captions & transcripts for accessibility
Responsive image example 
<picture>
  <source media="(max-width:600px)" srcset="/img/small.jpg">
  <img src="/img/large.jpg" alt="Demo image" loading="lazy">
</picture>

Accessibility & SEO

Simple rules that make big difference

Practical checklist

  • Always include alt on images
  • Use semantic headings & landmarks
  • Forms: use <label for="...">
  • Use rel="noopener" for external links opened in new tabs
  • Provide captions & transcripts for media
